## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Live OpenHolidays API → integration job runner | Untrusted upstream; rate-limit risk; schema drift risk. |
| Tag push by repo owner → release.yml | Trusted (push protections on main + tag protection — out of scope to enforce in YAML; managed in GitHub repo settings). |
| Sigstore/Fulcio (attestation) → release artifacts | Trusted (first-party OIDC + transparency-log-backed; D-11 chose this over cosign deliberately). |
