<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → consumer hook function | hook receives (req, resp, err) triple; the library does NOT trust the hook to behave correctly — panics propagate, body is not pre-read. |
| Synchronous hook contract | hook runs on the caller's goroutine; consumer owns any goroutine spawn. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-16 | Denial of Service | hook panic crashes the SDK / caller goroutine | accept (intentional) | D-90 explicit: panics propagate. Consumer wraps with `defer recover()` if needed (mirrors stdlib http.Handler convention). Silent recovery would hide bugs. `TestHookTransport_PanicPropagates` verifies. |
| T-04-17 | Information Disclosure | hook reads resp.Body and logs it at Info+ level | accept (documented) | RequestHookFunc godoc (declared in config.go by Plan 02) documents: "Hooks MUST NOT log resp.Body content above slog.LevelDebug (Pitfall LOG-1)." Consumer responsibility. |
| T-04-18 | Tampering | hook mutates the (req, resp, err) triple before downstream code sees them | accept (documented) | The hook receives the actual pointers. Mutating req.Header after RoundTripper chain returns has no effect (the request has already been sent). Mutating resp.Body BEFORE downstream consumes it WOULD corrupt the decoder. Document in RequestHookFunc godoc: "Hook MUST NOT read or modify resp.Body." Future hardening (v0.2) could pass an immutable view. |
| T-04-19 | Denial of Service | hook blocks indefinitely (e.g., synchronous log to a stalled syslog) | accept | Consumer's hook, consumer's problem. Document the synchronous contract in `WithRequestHook` godoc + Pitfall CONC-2 reference. Async variant deferred (CONTEXT.md `<deferred>`). |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No package installs — stdlib only. Per RESEARCH.md `Package Legitimacy Audit`. |
