<objective>
Close the W-01 Unicode case-fold bypass identified in Phase 1's verification follow-ups (`01-VERIFICATION.md` §follow_ups). The current `validateCountry` and `validateLanguage` implementations canonicalize via `strings.ToUpper` / `strings.ToLower` BEFORE the ASCII shape check; certain Unicode characters fold to ASCII letters (Kelvin sign U+212A → 'K', dotted/dotless I, long-s) and slip through the over-permissive post-fold len-2 check. The fix reorders: run a byte-level ASCII shape predicate against the ORIGINAL input first, then canonicalize.

Per CONTEXT.md D-32..D-34: this is a defect fix against the locked VALID-01 / VALID-04 contract (godoc already says "non-ASCII rejected"; implementation now matches). No new `Key Decisions` entry needed in PROJECT.md. Does not touch W-02 / W-03 / W-04 — those remain on the follow-ups backlog.

Purpose: Phase 3 endpoints (`PublicHolidays`, `SchoolHolidays`, `Subdivisions`) will call `validateCountry` / `validateLanguage` before issuing HTTP requests. The validators must be sound BEFORE that wiring lands; sequencing this plan in Wave 1 (parallel with Plan 01) ensures the fix is ready well in advance of Phase 3.

    // an ASCII letter (A-Z or a-z). Byte arithmetic (rather than unicode.IsLetter)

    func isTwoASCIILetters(s string) bool {
        if len(s) != 2 {

        }
